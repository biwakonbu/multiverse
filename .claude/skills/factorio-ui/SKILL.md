# Factorio風タイルベースUI設計原則

## いつ使うか

- 大量（10個以上）のAIエージェント/タスクを管理するUIを作る
- 全体像を俯瞰しながら個別の状態も把握したい
- タスク間の依存関係（フロー）を視覚化したい

## 設計思想

### なぜリストではなくタイルか

- リスト形式は10個程度なら有効だが、100個になると破綻する
- Factorioの工場UIのように、2D平面上に配置することで全体を俯瞰できる
- 3Dは魅力的だがディスプレイが2次元である以上、操作性に難がある

### コア原則

1. **タイル配置**: グリッド上にノードを整列配置し、秩序ある見た目を維持
2. **視覚的ステータス**: 色とアニメーションで状態を即座に識別（一瞥で把握）
3. **ズーム/パン**: 全体俯瞰から詳細まで自由に行き来
4. **フロー表現**: ノード間の依存関係を矢印で視覚化

### ステータスの視覚表現

- **実行中**: 目立つ色 + パルスアニメーション
- **成功**: 落ち着いた緑系
- **失敗**: 警告の赤系 + アイコン
- **待機中**: 控えめなグレー系

### レイアウト構成

- **メインエリア**: タイル配置のキャンバス（ズーム/パン可能）
- **サイドパネル**: 選択ノードの詳細表示
- **ツールバー**: グローバル操作、フィルター、Pool状態サマリ

## 実装時の注意

- ノード描画はCSS Transformが軽量で十分（Canvas/WebGLは過剰）
- 接続線はSVGが適切（ベジェ曲線、矢印マーカー）
- ドラッグ操作はPointer Eventsで統一（タッチ対応）
- 座標変換（画面↔グリッド↔キャンバス）を明確に分離
